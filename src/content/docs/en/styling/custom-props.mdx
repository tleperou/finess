Finess provides [`custom properties`](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties) to build consistent components with any framework. They map many of the css properties and leverage CSS features while taking into account both accessiblity and user's preferences.

## Borders

import BorderWidth from './../../../../starlight/fragments/custom-props/borders/borders.widths.astro'
import BorderRadius from './../../../../starlight/fragments/custom-props/borders/borders.radius.astro'

### `border-width`

<blockquote>The `border-width` [shorthand](https://developer.mozilla.org/en-US/docs/Web/CSS/Shorthand_properties) CSS property sets the width of an element's border. <cite>[MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/border-width)</cite></blockquote>

<BorderWidth />

<aside><p>`border-width` sets all flow-relative values, so specify the logical property (eg. `border-inline-width`) when used as a single unit. [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_logical_properties_and_values)</p></aside>

#### Usage

```css
/* block and inline shorthand */
blockquote {
  border-width: var(--border-width-1);
}

/* is the equivalent of */
blockquote {
  border-width: var(--border-block-width-1) var(--border-inline-width-1);
}

/* as a single unit */
blockquote {
  border-inline-start-style: solid;
  border-inline-start-color: var(--color-neutral-1);
  border-inline-width: var(--border-inline-width-3);
}

blockquote {
  /* ... */
  border-inline-start-width: var(--border-inline-width-3);
}
```

### `border-radius`

<blockquote>The `border-radius` CSS property rounds the corners of an element's outer border edge. <cite>[MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/border-radius)</cite></blockquote>

<BorderRadius />

<aside><p>`border-radius` sets all flow-relative values, so specify the logical property (eg. `border-start-start-radius`) when used as a single unit. [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_logical_properties_and_values)</p></aside>

#### Usage

```css
code {
  border-radius: var(--border-radius-1);
}

/* as a single unit */

code {
  border-start-start-radius: var(--border-start-end-radius-1);
}
```

### `font-size`

<blockquote>The `font-size` CSS property sets the size of the font relatively to the dimensions of the viewport. <cite>[MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/font-size)</cite></blockquote>

<FontSizes />

<aside><p>The typography is fluid, that means the `font-size` varies with the viewport's size â€“ in the respect of the [WCAG SC 1.4.4](https://www.w3.org/WAI/WCAG21/Understanding/resize-text.html).</p></aside>

#### Usage

```css
code {
  font-size: var(--font-size-1);
}
```

### `font-weight`

<blockquote>The `font-weight` CSS property sets the weight (or boldness) of the font. The weights available depend on the [`font-family`](https://developer.mozilla.org/en-US/docs/Web/CSS/font-family) that is currently set. <cite>[MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/font-weight)</cite></blockquote>

<FontWeight />

#### Usage

```css
code {
  font-weight: var(--font-weight-1);
}
```

### `letter-spacing`

The `letter-spacing` CSS property sets the horizontal spacing behavior between text characters.

<LetterSpacing />

#### Usage

```css

  letter-spacing: var(--letter-spacing-1);
}
```


### `line-height`

<blockquote>The `line-height` CSS property sets the height of a line box. It's commonly used to set the distance between lines of text. On block-level elements, it specifies the minimum height of line boxes within the element. <cite>[MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/line-height)</cite></blockquote>

<LineHeight />

#### Usage

```css
code {
  line-height: var(--line-height-1);
}
```

## Media queries

import MediaQueryDevice from './../../../../starlight/fragments/custom-props/media/media.device.astro'
import MediaQueryPrefers from './../../../../starlight/fragments/custom-props/media/media.prefers.astro'
import MediaQueryBreakpoint from './../../../../starlight/fragments/custom-props/media/media.breakpoints.astro'

<blockquote>
  <p>Media queries allow you to apply CSS styles depending on a device's media type, features and user preferences. Many custom props already leverage the media queries <cite>[MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_media_queries/Using_media_queries)</cite></p>
</blockquote>

<aside>
  Using media queries requires `custom-media` syntax, available with this [PostCSS plugin](https://github.com/csstools/postcss-plugins/tree/main/plugins/postcss-custom-media).
</aside>

### Device

<MediaQueryDevice />

#### Usage

```css
:root {
  --my-color: #21307a;

  @media (--hdr) {
    --my-color: color(display-p3 0.1417 0.1865 0.461)
  }
}
```

### User preferences

<MediaQueryPrefers />

#### Usage

```css
:root {
  --transition-timing-1: 200ms;

  @media (--mation-reduced) {
    --transition-timing-1: 60ms;
  }
}
```

<aside>
  This is the way [`--transition-timing-{1-3}`](#transition-timing) and [`--opacity-{1-3}`](#opacity) are taking advantage of the user preferences.
</aside>

### Breakpoints

<MediaQueryBreakpoint />

#### Usage

```css
.my-layout {
  display: grid;
  grid-template-columns: `1fr`;

  @media (--breakpoint-md-n-above) {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

## Spacings

import Gaps from './../../../../starlight/fragments/custom-props/spacings/spacings.gaps.astro'
import Margins from './../../../../starlight/fragments/custom-props/spacings/spacings.margins.astro'
import Paddings from './../../../../starlight/fragments/custom-props/spacings/spacings.paddings.astro'

Spacings are relative to the element's [`font-size`](https://developer.mozilla.org/en-US/docs/Web/CSS/font-size) and [`line-height`](https://developer.mozilla.org/en-US/docs/Web/CSS/line-height) based upon the average glyph's dimensions. It also vary with the height, width and orientation of the viewport.

### `gap`

<aside><p>`gap` sets all flow-relative values, so specify `row-gap` and `column-gap` when used as a single unit.</p></aside>

<Gaps>
  <p slot="row">The [`row-gap`](https://developer.mozilla.org/en-US/docs/Web/CSS/row-gap) CSS property sets the size of the gap between an element's rows.</p>
  <p slot="column">The [`column-gap`](https://developer.mozilla.org/en-US/docs/Web/CSS/column-gap) CSS property sets the size of the gap between an element's columns.</p>
  <p slot="shorthand">The <a href="https://developer.mozilla.org/en-US/docs/Web/CSS/row-gap"><code>row-gap</code></a> CSS property sets the size of the gap between an element's rows.</p>
</Gaps>

#### Usage

```css
/* as a shorthand */
.card {
  display: grid;
  gap: var(--gap-1);
}

/* equivalent of */
.card {
  display: grid;
  gap: var(--column-gap-1) var(--row-gap-1);
}

/* as a single value */
.card {
  display: grid;
  row-gap: var(--row-gap-1);
  column-gap: var(--column-gap-3);
}
```

<aside><p>`margin` sets all flow-relative values, so specify the logical property (eg. `margin-inline`) when used as a single unit. [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_logical_properties_and_values)</p></aside>

<Margins />
